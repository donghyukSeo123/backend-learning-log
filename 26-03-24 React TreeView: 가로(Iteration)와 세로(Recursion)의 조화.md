# 🌳 React TreeView: 가로(Iteration)와 세로(Recursion)의 조화

`ContentTree.js`에서 데이터를 화면에 그리는 원리는 **'가로로 훑기'**와 **'세로로 파고들기'** 두 가지 메커니즘이 합쳐진 결과입니다.

## 1. 가로 방향: `map` (반복문)
서버에서 받아온 최상위 리스트(프로젝트들)를 하나씩 꺼내는 과정입니다.

- **코드**: `treeData.map((node) => renderTreeNodes(node))`
- **역할**: "내 서랍에 프로젝트가 3개 있네? 하나씩 꺼내서 검사받으러 보내야지."
- **특징**: 데이터의 **개수**만큼 옆으로(가로로) 반복합니다.



---

## 2. 세로 방향: `renderTreeNodes` (재귀함수)
꺼내진 하나의 프로젝트 안에서 실제 파일이 나올 때까지 끝까지 파고드는 과정입니다.

- **코드**: `if (node.children) { node.children.map(child => renderTreeNodes(child)) }`
- **역할**: "이 프로젝트 폴더 안에 또 폴더나 파일이 있네? 없을 때까지 계속 안으로 들어가보자."
- **특징**: 데이터의 **깊이(Depth)**만큼 아래로(세로로) 파고듭니다.



---

## 3. 전체 동작 프로세스 (가로 x 세로)

| 단계 | 동작 방식 | 비유 |
| :--- | :--- | :--- |
| **1단계 (가로)** | `treeData.map` 실행 | 거실에 놓인 **상자들(프로젝트)**을 하나씩 확인한다. |
| **2단계 (세로)** | `renderTreeNodes` 진입 | 상자를 열었는데 **작은 상자(하위 폴더)**가 나오면 또 연다. |
| **3단계 (도착)** | `return <TreeItem />` | 마침내 **내용물(파일)**이 나오면 화면에 그림을 그린다. |

---

## 4. 왜 이 구조가 강력한가? (Scalability)

1. **무한 계층 지원**: 폴더 구조가 2단이든 100단이든 코드를 수정할 필요가 없습니다. (재귀가 알아서 끝까지 가기 때문)
2. **유연한 데이터 대응**: 프로젝트가 1개일 때나 1,000개일 때나 `map` 한 줄로 대응이 가능합니다.
3. **유지보수 용이**: 파일 모양을 바꾸고 싶으면 재귀 함수의 가장 마지막 `return` 부분(파일 노드)만 수정하면 모든 파일에 적용됩니다.
