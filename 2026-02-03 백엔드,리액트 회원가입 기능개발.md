## 2026-02-03 – 리액트(React) 입문: JS와는 달랐던 새로운 패러다임 정리

## 오늘 한 것 (React Specific)
- **선언적 UI 구현**: JS처럼 `document.getElementById`로 일일이 찾지 않고, 데이터(`State`)만 바꾸면 화면이 알아서 갱신되는 구조 적용.
- **컴포넌트 기반 설계**: `MDInput`처럼 자주 쓰는 요소를 모듈화하고, 레고 블록 조립하듯 회원가입 폼 구성.
- **상태 동기화 로직**: 비밀번호 일치 여부를 수동으로 계산하지 않고, 상태값 비교를 통해 리액트가 자동 대조하도록 설계.
- **비동기 렌더링 제어**: 메일 발송 비동기 작업 종료 후, 결과 데이터를 화면에 즉각 꽂아주는 리액트식 처리 실습.

## 오늘 새로웠던 점 (JS vs React)
- **State vs Variable**: 단순 변수와 달리 `useState`는 값이 변하는 순간 리액트가 감지하여 화면을 다시 그린다는 점.
- **Props (전달)**: 자식 컴포넌트에 데이터를 넘길 때 HTML 속성처럼 `{...props}`로 쓱 집어넣는 방식의 편리함.
- **JSX 문법**: JS 코드 안에서 HTML 태그를 바로 작성하는 방식이 로직과 UI를 연결하기에 훨씬 직관적임.

---

## 🧠 오늘의 개념 (JS 개발자가 본 리액트)

### ⚡ 명령형(JS) vs 선언적(React)
- **JS**: "버튼 찾아서 색깔 바꿔!" (일일이 명령)
- **React**: "상태가 이러면 버튼은 이 색깔이야." (상태만 정의)
- **통찰**: 코드가 간결해지고 화면을 직접 건드리지 않아도 된다는 해방감.

### 🔄 데이터 흐름의 단방향성
- 데이터가 부모에서 자식으로만 흐르기 때문에 어디서 문제가 생겼는지 추적이 훨씬 명확함.

### 📦 컴포넌트 재사용
- 똑같은 입력창을 여러 개 쓸 때, JS처럼 복붙하지 않고 컴포넌트 하나로 재사용하여 유지보수성 극대화.

---

## 📌 요약
- 자바스크립트 숙련도를 바탕으로 리액트의 **상태 관리와 컴포넌트 구조**를 빠르게 습득함.
- 특히 메일 인증 비동기 통신 결과를 화면에 반영할 때, 리액트의 `State` 방식이 JS보다 훨씬 효율적임을 확인.
