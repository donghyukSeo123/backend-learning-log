# [배포 학습 - Part 5] AWS EC2 인프라 설정 및 프리티어 100% 활용 실전 가이드

이 파트에서는 AWS 프리티어 인스턴스(t2/t3.micro)의 1GB 메모리 극복법인 가상 메모리(Swap) 설정부터 시작하여, 서버 보안 그룹 및 실무에서 사용하는 관리 명령어들을 총망라하여 학습합니다.

---

## 1. t2/t3.micro 가상 메모리(Swap) 설정법
AWS 프리티어 서버(t2/t3.micro)는 물리 메모리(RAM)가 오직 **1GB**입니다. 이 상태에서 백엔드 빌드(`gradle build`)와 프론트 빌드(`npm run build`)가 겹치거나 동작하면 램이 모자라 서버가 통째로 굳어버리거나 먹통이 됩니다. 

이를 방지하기 위해 가상 서버의 하드디스크(SSD)의 일부를 램(Memory)처럼 빌려 쓰는 **스왑(Swap) 메모리** 설정이 프리티어 환경에서는 **필수적**입니다.

```bash
# 1. 2GB 크기의 빈 스왑용 파일 할당
sudo fallocate -l 2G /swapfile

# 2. 파일 권한을 소유자(root)만 읽고 쓸 수 있도록 제한 (보안 필수)
sudo chmod 600 /swapfile

# 3. 이 파일을 스왑 메모리 영역으로 포맷
sudo mkswap /swapfile

# 4. 스왑 영역 활성화
sudo swapon /swapfile

# 5. 서버가 재부팅될 때마다 스왑 메모리가 영구적으로 활성화되도록 fstab 파일에 설정 한 줄 추가
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

## 2. 보안 그룹(Security Group) 포트 설계
보안 그룹은 AWS 가상 컴퓨터의 **외곽 방화벽**입니다. 우리는 포트 노출을 최소화하여 해커의 접근 경로를 차단했습니다.

*   **22 (SSH)**: 서버 원격 관리 목적. 본인 IP만 허용하도록 지정하는 것이 정석이나 유동 IP 환경일 경우 위치 무관(`0.0.0.0/0`)으로 열어둔 채 Key 파일(`.pem`)을 철저히 보안 보관합니다.
*   **80 (HTTP)**: 전 세계 사용자들이 브라우저를 통해 리액트 화면에 진입하는 대문입니다. 반드시 전체 허용(`0.0.0.0/0`) 해야 합니다.
*   **443 (HTTPS)**: 추후 보안 인증서(SSL/TLS) 연동을 위해 미리 열어둡니다.
*   **5432 (PostgreSQL)**: 로컬 DB 클라이언트 툴(DBeaver, DataGrip 등)로 서버 DB 테이블 구조나 데이터를 원격 점검하기 위해 임시로 오픈할 수 있습니다.

## 3. 서버 배포 실전 명령어 시트 (Quick Reference)

서버 최초 구동 및 코드 업데이트 시 터미널에서 실행하게 될 주요 커맨드 모음입니다.

```bash
# ----------------------------------------------------
# 1. 최초 배포 시 구동 순서
# ----------------------------------------------------
# 프로젝트 다운로드
git clone [본인의 깃허브 저장소 주소]
cd Insajang

# 환경 변수 비밀 설정 파일 작성
cp .env.example .env
nano .env  # 실제 메일 패스워드와 Gemini Key 입력

# 도커 컴포즈 무중단 기동 및 백그라운드 구동 (-d)
docker compose up --build -d

# ----------------------------------------------------
# 2. 코드 수정 후 서버 업데이트 순서 (배포 자동화)
# ----------------------------------------------------
# 코드 최신화
git pull origin main

# 특정 컨테이너만 재빌드 및 재시작 (예: 백엔드가 수정된 경우)
docker compose up --build -d insajang-backend

# ----------------------------------------------------
# 3. 디버깅 및 상태 검사 명령어
# ----------------------------------------------------
# 실행 중인 컨테이너 프로세스 및 포트 확인
docker compose ps

# 백엔드 로그 실시간 모니터링
docker compose logs -f insajang-backend

# 파이썬 AI 로그 실시간 모니터링
docker compose logs -f insajang-python

# DB 컨테이너 쉘 접속
docker exec -it insajang-db psql -U postgres -d insajang
```
