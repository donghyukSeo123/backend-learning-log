# 2026-01-27 – 인사장프로젝트: 리액트 UI 로직 및 상태 관리
오늘 한 것
MDInput 활용 비밀번호 확인 UI 구현

useState를 이용한 실시간 비밀번호 일치 여부 검증 로직 연동

부모-자식 컴포넌트 간 데이터 전달(Props) 구조 파악

오늘 알게 된 것
useState: 상태가 변하면 화면을 다시 그리는 리액트의 핵심 훅.

Props: 컴포넌트 간 데이터를 전달하는 방식. {...props}로 모든 속성 일괄 전달 가능.

Event Handling: onChange 등을 통해 사용자의 입력을 데이터로 변환하는 과정.

🧠 오늘의 개념
📦 Props (Properties)
부모 컴포넌트가 자식 컴포넌트에게 넘겨주는 데이터 묶음.

특징: 자식은 전달받은 값을 읽기만 할 수 있으며(Immutable), 이를 통해 컴포넌트의 재사용성이 결정됨.

⚡ useState
상태 변수: 현재의 데이터 값.

설정 함수(Setter): 값을 변경하고 화면 리렌더링을 트리거함.

인사장님 비유: "데이터가 바뀌면 화면도 즉시 바뀌게 만드는 실시간 엔진."

🔄 Event Handling (onChange)
사용자의 행동(타이핑, 클릭 등)을 감지하여 로직을 실행하는 통로.

e.target.value를 통해 입력된 값을 추출하고, 이를 Setter 함수에 전달하여 상태를 업데이트함.

📌 요약
Props는 데이터 전달, Event Handling은 데이터 추출, useState는 화면 반영을 담당함.

이 구조를 이해하면 리액트의 데이터 흐름(Data Flow) 전체를 파악할 수 있음.
