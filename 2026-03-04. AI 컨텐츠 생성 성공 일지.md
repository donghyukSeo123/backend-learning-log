## 🚀 1. API 할당량 초과(429 Error) 및 모델 최적화 및 통신 성공 
AI 호출 중 발생한 `429 Too Many Requests` 에러를 해결하고, 우리 환경에 가장 적합한 모델을 확정했습니다.

### 🔍 문제 원인
* Gemini 2.0/3.1 등 최신 프리뷰 모델은 무료 티어의 분당 호출 제한(RPM)이 매우 낮음.
* 짧은 시간 내 반복 요청 시 구글 서버에서 일시적으로 차단.

### ✅ 해결 방안
* **모델 변경**: 가장 안정적이고 속도가 빠른 `models/gemini-flash-latest`를 메인 엔진으로 채택.


## 🛠️ 2. 리액트-스프링-파이썬 타임아웃 동기화
AI가 고퀄리티 컨텐츠를 생성하는 시간(약 10~20초)을 시스템이 견디지 못하고 끊어버리는 문제를 해결했습니다.

### ⚙️ 설정 변경 (React / Axios)
기본 타임아웃을 **60초**로 대폭 늘려 AI가 답변을 마칠 때까지 연결을 유지합니다.

```javascript
// 프론트엔드 API 호출부
const response = await API.post("/api/contents/createContent", {
    project_id: selectedProjectId,
    title: title,
    user_input: inputText,
    content_type: type
}, { 
    timeout: 60000 // 🚀 60초 타임아웃 설정 (반드시 데이터 객체 뒤에 설정 객체로 전달)
});
