# [배포 학습 - Part 3] React 프론트엔드 (`insajang-frontend`) 빌드 및 경량화

이 파트에서는 Node.js 개발서버 환경을 배포 환경에서 걷어내고, 초경량 Nginx와 리액트 빌드 파일만 조화시켜 배포하는 과정을 학습합니다.

---

## 1. Dockerfile 코드 분석
[Dockerfile](file:///c:/workspace/Insajang/insajang-frontend/Dockerfile)

```dockerfile
# 1단계: 빌드 스테이지 (Node.js를 사용하여 React 빌드)
FROM node:18-slim AS build
WORKDIR /app

# npm 패키지 설치
COPY package*.json ./
RUN npm install

# 소스 복사 및 정적 파일 빌드
COPY . .
RUN npm run build

# 2단계: 실행 스테이지 (Nginx를 사용하여 정적 HTML 서빙 및 리버스 프록시 작동)
FROM nginx:alpine
# 빌드된 결과물을 Nginx 홈 디렉토리로 이동
COPY --from=build /app/build /usr/share/nginx/html
# 작성한 nginx.conf 설정을 컨테이너 내부로 덮어쓰기
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

## 2. 핵심 이론 및 원리 학습

### ① 프론트엔드 배포의 패러다임 전환
*   로컬 개발 환경에서는 `npm start`를 치면 Node.js가 개발 서버를 임시로 띄워 소스코드 변경을 감시하고 컴파일합니다.
*   **주의**: 프로덕션(배포) 환경에서는 Node.js 서버를 직접 돌리지 않습니다. Node.js는 싱글 스레드라 트래픽 처리에 비효율적이고 메모리를 아주 많이 먹기 때문입니다.
*   **정석적인 해결**: 1단계에서 Node.js를 이용해 브라우저가 직접 읽을 수 있는 순수한 정적 파일 뭉치(HTML, JS, CSS, 이미지)로 완전히 구워버립니다(`npm run build`). 그리고 2단계에서는 전 세계에서 가장 빠르고 가벼운 정적 파일 배포 서버인 **Nginx**를 띄워 이 구워진 파일들만 사용자에게 반환합니다.

### ② `package*.json` 선복사의 효과
*   React에서 가장 용량을 크게 차지하는 폴더가 바로 `node_modules`입니다.
*   소스코드가 한 자만 바뀌어도 수백 개의 외부 패키지를 다시 다운로드하는 사태를 피하기 위해, `package.json`과 `package-lock.json`만 선복사하여 `npm install`을 먼저 실행(캐싱)한 뒤, 소스코드 복사(`COPY . .`) 및 빌드를 태웁니다.
