# # 2026-01-27 – 인사장프로젝트: 실시간 UI 로직 및 보안/환경 설정 최적화

## 오늘 한 것
- `MDInput` 활용 비밀번호 확인 UI 구현
- `useState`를 이용한 실시간 비밀번호 일치 여부 검증 로직 연동
- 부모-자식 컴포넌트 간 데이터 전달(`Props`) 구조 파악
- 프로젝트 보안 및 가독성을 위한 루트 경로 설정(`baseUrl`) 최적화
- 사용자 경험(UX) 개선을 위한 인터페이스 문구 일괄 변경

## 오늘 알게 된 것
- **useState**: 상태가 변하면 화면을 다시 그리는 리액트의 핵심 훅.
- **Props**: 컴포넌트 간 데이터를 전달하는 방식. `{...props}`로 모든 속성 일괄 전달 가능.
- **Event Handling**: `onChange` 등을 통해 사용자의 입력을 데이터로 변환하는 과정.
- **Root Path Config**: 내부 구조를 보호하고 경로 노출을 최소화하는 보안 설정의 기초.

---

## 🧠 오늘의 개념

### 🛡️ 루트 경로 및 보안 설정 (`jsconfig.json`)
- **설정 내용**: `baseUrl: src` 설정을 통해 프로젝트 내부 구조를 체계화함.
- **보안적 측면**: 상대 경로(`../../`) 사용 시 노출될 수 있는 파일 계층 구조를 가리고, 내부 모듈 접근을 엄격하게 관리하는 기초가 됨.

### ✍️ 인터페이스 문구 변경 (UI Text)
- 서비스 컨셉에 맞춰 딱딱한 문구들을 직관적으로 수정하여 사용자가 기능을 명확히 이해하도록 UX 개선.

### 📦 Props (Properties)
- 부모 컴포넌트가 자식 컴포넌트에게 넘겨주는 데이터 묶음.
- **특징**: 자식은 전달받은 값을 읽기만 할 수 있으며(Immutable), 이를 통해 컴포넌트의 재사용성이 결정됨.

### ⚡ useState
- **상태 변수**: 현재의 데이터 값.
- **설정 함수(Setter)**: 값을 변경하고 화면 리렌더링을 트리거함.

### 🔄 Event Handling (onChange)
- 사용자의 행동(타이핑, 클릭 등)을 감지하여 로직을 실행하는 통로.
- `e.target.value`를 통해 입력된 값을 추출하고, 이를 `Setter` 함수에 전달하여 상태를 업데이트함.



---

### 📌 요약
- **Props**는 데이터 전달, **Event Handling**은 데이터 추출, **useState**는 화면 반영을 담당함.
- **보안 설정**으로 프로젝트 기초를 다지고, 이 구조를 통해 리액트의 데이터 흐름(Data Flow) 전체를 파악함.
